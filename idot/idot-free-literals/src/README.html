Extension of ιDOT with Free Literals

This directory contains an extension of ιDOT which allows literals to be allocated on free subheap. Details of the extension can be found in our paper.

Documentation

The documentation for the repository can be accessed form the table of contents, which lists links to pretty printed Coq files, but the raw ".v" source files are also available in this directory. In the pretty-printed versions, the proof scripts are hidden by default, but you may click on "Show Proofs" at the top of the page to display all the proofs, or click under the Lemma, Corollary, or Theorem statements to display their proofs.

Differences from the Base Calculs

The main difference from the base calclus is in the Initalization System. The extension features a new rule for giving initialization types to literal bindings init_lit_free, which allocates non-committed variables to show up unbound in literals.

Additionally, we inline the subtyping relation into the initialization lookup relation (init_var). However, this is a superficial difference and the lookup rules in the base calculus are equilavent to the ones used here.

Even with the new initialization rules, the type safety lemmas still hold.
