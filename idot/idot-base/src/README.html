ιDOT: A DOT Calculus with Safe Object Initialization

Our paper presents ιDOT calculus, which extends the κDOT calculus of Kabir and Lhoták with a type and effect system object to ensure null-safety and makes fields non-lazy. κDOT is itself an extension of the WadlerFest DOT calculus of Amin el al. (2016). This repository contains a formalization of the type-safety proof of the calculus in Coq.

The type-safety proof for the ιDOT caluclus extends the Rapoport el al. (2017) type safety proof for WadlerFest DOT.

Documentation

The documentation for the repository can be accessed form the table of contents, which lists links to pretty printed Coq files, but the raw ".v" source files are also available in this directory. In the pretty-printed versions, the proof scripts are hidden by default, but you may click on "Show Proofs" at the top of the page to display all the proofs, or click under the Lemma, Corollary, or Theorem statements to display their proofs.

Differences from the Paper Version

The base calculus presented in our paper is simpler than the one we proved sound in this repository. The Coq version in this repository allows for an additional initialization type "unspecified" and allows subtyping between the other initialization types and "unspecified". Since there is no subtyping, in the paper version can get away without using the init_binds_sub condition for the Substution lemma for the initialization system (well_init_subst_term). Additionally, configuration typing in the paper version does not use the inert_inits condition since initialization contexts for the paper version are always inert.
